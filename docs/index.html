How it works

Same sensory profile as a person with visual impairment

A person with visual impairment navigating a collapsed building has no camera feed — only proximity awareness, spatial memory, and touch. DEBRIS simulates exactly this: a 71-dimensional sensor array with no visual input, trained via PPO to find the exit under falling debris and crumbling ceilings.

◈

Spatial sensors only

71-dim observation vector: position, velocity, debris proximity, heading to exit. No RGB pixels. No depth camera. Pure spatial awareness — the same information available to a cane user with spatial memory.

◉

Debris avoidance

5 primary falling slabs (18–30 kg) and 5 scatter shrapnel bodies, ceiling crumble waves, fire hazard zones, and wall collapse — all active simultaneously in the final stage. Policy learns to predict and dodge from velocity alone.

◎

Disaster-scale training

4-stage curriculum from open corridor navigation to full chaos. 4.3 M training steps on A100 GPU via Modal, 16 parallel envs, PPO with GAE — no human demonstrations, only physics reward signals.

Curriculum

Four progressive stages

Hazards introduced incrementally so the policy first masters navigation, then adds avoidance. Each stage loads weights from the previous checkpoint.

# Stage Primary debris Scatter Crumble Steps End reward
1 static_only 00—600k +8.5
2 debris_only 30—1.0M −2.8
3 debris_and_scatter 53—1.2M +0.17
4 full_chaos 55 ✓ 1.5M −10.9

Physics method: MuJoCo weld equality constraints released at runtime (eq_active[i] = 0), dropping 180–420 kg concrete slabs under gravity. 9 weld constraints total, triggered by elapsed steps or agent x-position crossing a threshold.

Observation Space

71-dimensional · no vision

Every input the agent receives is also available to a person using spatial memory and a white cane. No RGB pixels. No depth map. The policy navigates purely from physics-derived proximity and velocity data.

Indices Name Dim Description
[0:3] agent_pos 3 xyz world frame position
[3:6] agent_vel 3 vx vy vz velocity
[6:9] agent_facing 3 forward unit vector (body x-axis)
[9:12] goal_relative_pos 3 goal − agent in world frame
[12:27] primary_debris_pos 15 5 bodies × xyz — falling slabs (18–30 kg)
[27:32] primary_debris_vz 5 vertical fall velocity — negative = incoming
[32:47] scatter_debris_pos 15 5 scatter shrapnel bodies × xyz
[47:52] scatter_debris_vz 5 scatter vertical fall velocities
[52:57] primary_debris_vx 5 lateral x velocity — predicts landing zone
[57:62] primary_debris_vy 5 lateral y velocity — predicts landing zone
[62] time_remaining_norm 1 1.0 = episode start · 0.0 = timeout
[63] heading_to_goal_angle 1 radians, 0 = facing goal directly
[64:69] ceil_broken_flags 5 1 = weld broken, tile now falling
[69] fire_zone_flag 1 1 = agent within 0.65 m of fire zone
[70] collision_flag 1 1 = debris contact force above threshold

Action space · 3-dim: vx ∈ [−1.5, 1.5] m/s · vy ∈ [−1.0, 1.0] m/s · ωz ∈ [−1.0, 1.0] rad/s
Policy output: tanh activation → scaled to physical ranges. MLP actor-critic: Linear(71→256)→LayerNorm→Tanh ×2. PPO: clip=0.2, GAE λ=0.95, γ=0.99, 16 parallel envs.
